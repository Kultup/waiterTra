# Шаблон сценарію гри (Scenario Template)

## Формат файлу

Шаблони сценаріїв зберігаються у форматі JSON з такою структурою:

```json
{
  "$schema": "serviq-scenario-template-v1",
  "title": "Назва сценарію",
  "description": "Короткий опис сценарію",
  "targetCity": "Місто (опціонально)",
  "characters": [
    {
      "name": "Ім'я персонажа",
      "avatar": "🧑‍🍳",
      "color": "#ff6d5a",
      "description": "Опис персонажа"
    }
  ],
  "nodes": [
    {
      "text": "Текст сцени",
      "speaker": "Ім'я персонажа (опціонально)",
      "choices": [
        {
          "text": "Текст вибору",
          "nextNode": "Текст наступної сцени (null якщо кінець)",
          "isWin": false,
          "result": "Результат фіналу"
        }
      ],
      "x": 100,
      "y": 100
    }
  ],
  "startNode": "Текст стартової сцени"
}
```

## Поля шаблону

### Обов'язкові поля

| Поле | Тип | Опис |
|------|-----|------|
| `title` | string | Назва сценарію |
| `nodes` | array[] | Масив вузлів (сцен) |
| `startNode` | string | Текст стартової сцени (має співпадати з одним з вузлів) |

### Опціональні поля

| Поле | Тип | Опис |
|------|-----|------|
| `$schema` | string | Версія схеми шаблону |
| `description` | string | Опис сценарію |
| `targetCity` | string | Місто для якого призначений сценарій |
| `characters` | array[] | Масив персонажів |

## Структура персонажа

```json
{
  "name": "Ім'я персонажа",
  "avatar": "🧑‍🍳",
  "color": "#ff6d5a",
  "description": "Опис"
}
```

| Поле | Тип | Опис |
|------|-----|------|
| `name` | string | Ім'я персонажа (унікальне) |
| `avatar` | string | Emoji-аватар |
| `color` | string | HEX колір |
| `description` | string | Короткий опис |

## Структура вузла

```json
{
  "text": "Текст сцени",
  "speaker": "Ім'я персонажа",
  "choices": [...],
  "x": 100,
  "y": 100
}
```

| Поле | Тип | Опис |
|------|-----|------|
| `text` | string | Текст сцени (унікальний ідентифікатор) |
| `speaker` | string | Ім'я персонажа який говорить (співпадає з `name` з `characters`) |
| `choices` | array[] | Масив виборів |
| `x` | number | Позиція X на канвасі |
| `y` | number | Позиція Y на канвасі |

## Структура вибору

```json
{
  "text": "Текст кнопки вибору",
  "nextNode": "Текст наступної сцени",
  "isWin": true,
  "result": "Текст фіналу"
}
```

| Поле | Тип | Опис |
|------|-----|------|
| `text` | string | Текст кнопки вибору |
| `nextNode` | string\|null | Текст сцени куди веде вибір (null = кінець гри) |
| `isWin` | boolean | Чи є цей фінал переможним |
| `result` | string | Текст результату (для фіналів) |

## Як використовувати

### Експорт сценарію

1. Відкрийте сценарій у візуальному редакторі
2. Натисніть кнопку 📤 вгорі редактора
3. Файл шаблону завантажиться на ваш комп'ютер

**АБО** прямо зі списку сценаріїв:
1. На головній сторінці знайдіть потрібний сценарій
2. Натисніть 📤 у картці сценарію
3. Файл завантажиться

### Імпорт сценарію

**Імпорт JSON:**
1. На головній сторінці редактора натисніть "📥 Імпорт"
2. Оберіть JSON-файл шаблону
3. Перегляньте попередній перегляд
4. Натисніть "Імпортувати"
5. Відредагуйте сценарій за потреби та збережіть

**Імпорт Excel (.xlsx):**
1. Підготуйте Excel файл з колонками (порядок не важливий):
   - **Текст сцени** (обов'язково) - текст сцени
   - **Персонаж** (опціонально) - хто говорить
   - **Вибір** (опціонально) - текст вибору
   - **Наступна сцена** (опціонально) - куди веде вибір
   - **Перемога** (опціонально) - TRUE/FALSE для фіналів
   - **Результат** (опціонально) - текст фіналу
2. Натисніть "📥 Імпорт"
3. Оберіть Excel файл
4. Сценарій автоматично створиться з даних

### Приклад Excel таблиці:

| Текс сцени | Персонаж | Вибір | Наступна сцена | Перемога | Результат |
|------------|----------|-------|----------------|----------|-----------|
| Гість заходить у ресторан... | | Привітати гостя | Гість посміхається... | | |
| Гість заходить у ресторан... | | Ігнорувати гостя | | FALSE | Гість йде ображений |
| Гість посміхається... | Офіціант | Запропонувати столик | | TRUE | Гість задоволений |

**Готові файли для тестування:**
- `templates/excel_scenario_template.xlsx` - Excel шаблон
- `templates/excel_scenario_template.csv` - CSV версія
- `templates/restaurant_greeting_template.json` - JSON шаблон

## Валідація

Під час імпорту перевіряється:
- ✅ Наявність поля `title`
- ✅ Наявність масиву `nodes` (хоча б 1 вузол)
- ✅ Наявність поля `startNode`
- ✅ Співпадіння `startNode` з текстом одного з вузлів
- ✅ Коректність JSON-формату

## Примітки

- Імена персонажів мають бути унікальними в межах `characters`
- Текст сцени (`text`) використовується як ідентифікатор для зв'язків між вузлами
- Поля `x` та `y` визначають початкове розташування на канвасі (опціонально)
- Якщо `nextNode` не співпадає ні з одним вузлом, зв'язок буде розірвано
